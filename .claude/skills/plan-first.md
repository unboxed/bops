# Plan-First Template

Use for any task beyond simple fixes.

```
## Task
[One sentence description]

## Success Criteria
- [What must be true for this to be considered done]

## Out of Scope
- [Explicitly not doing]

## Context
- Relevant files: [list paths]
- Existing patterns: [how similar things are done]
- Constraints: [must/must not]

## Approach
1. [First change]
2. [Second change]
3. [etc.]

## Verification
- [ ] Tests pass
- [ ] Linting passes
- [ ] Manual check: [what to verify]
```

## When to Use

- Adding new features
- Refactoring existing code
- Fixing non-obvious bugs
- Any change touching 3+ files

## When to Skip

- Typo fixes
- Single-line changes
- Adding a log statement
